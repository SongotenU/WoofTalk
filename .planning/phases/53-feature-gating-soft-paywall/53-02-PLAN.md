# Wave 2: Translation Gating

## Objective
Modify translation request flow to check daily limit (3 translations/day) for free users.
Block translation attempts beyond limit with non-blocking upgrade prompt linking to paywall.
Update history retrieval to return only last 10 items for free users; premium users get full history.
Display lock icon on translation button when free tier limit reached.

## Tasks

### Translation Request Flow (All Platforms)
- [ ] Before making translation request, check EntitlementManager.isPremium
- [ ] If !isPremium, check EntitlementManager.dailyTranslationsUsed >= 3
- [ ] If limit reached, show non-blocking upgrade prompt (do not make request)
- [ ] If limit not reached, proceed with translation request
- [ ] After successful translation, update dailyTranslationsUsed (via EntitlementManager or lightweight endpoint)

### History Retrieval (All Platforms)
- [ ] When fetching translation history, check EntitlementManager.isPremium
- [ ] If !isPremium, limit results to last 10 items (client-side filter)
- [ ] If isPremium, return full history
- [ ] Update history UI to indicate when history is truncated for free users

### UI Updates (All Platforms)
- [ ] On translation button: when dailyTranslationsUsed >= 3 and !isPremium, show lock icon overlay
- [ ] On translation button: when tapped and limit reached, show upgrade prompt instead of making request
- [ ] Upgrade prompt: non-blocking modal/bottom sheet with "Upgrade" button linking to paywall
- [ ] Upgrade prompt: dismissible by user
- [ ] Upgrade prompt: includes messaging about free tier limit (3 translations/day)

## Success Criteria
- Free users limited to 3 translation requests per day (client-side prevention)
- After 3rd translation, free users see non-blocking upgrade prompt linking to paywall
- Translation button shows lock icon when free tier limit reached
- Free users see only last 10 history items; premium users see full history
- Premium users have unlimited translations and full history access
