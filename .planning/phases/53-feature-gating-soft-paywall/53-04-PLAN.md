# Wave 4: Watch App & Cross-Platform Sync

## Objective
Implement watch app subscription status inheritance from paired phone (no separate purchase required).
Ensure watch app respects same feature gating and limits as phone.
Test cross-platform entitlement sync (purchase on phone unlocks premium on watch immediately).

## Tasks

### Watch App Implementation (iOS/watchOS)
- [ ] Verify Watch app uses same Supabase auth.uid as paired iPhone
- [ ] Verify Watch app shares EntitlementManager logic with iPhone (via shared framework or Watch Connectivity)
- [ ] Ensure Watch app reads isPremium, isTrialActive, dailyTranslationsUsed from EntitlementManager
- [ ] Implement feature gating on Watch app using same logic as iPhone:
  - Translation limit: 3/day for free users
  - History limit: last 10 items for free users
  - Premium features (AI translation, etc.) locked for free users
- [ ] Ensure Watch app shows same upgrade prompts and lock icons as iPhone
- [ ] Test that Watch app inherits subscription status immediately when iPhone purchases

### Cross-Platform Entitlement Sync Testing
- [ ] Test scenario: User is free tier on iPhone, purchases premium → verify Watch app immediately gets premium access
- [ ] Test scenario: User is premium on iPhone, cancels subscription → verify Watch app immediately reverts to free tier
- [ ] Test scenario: User purchases on Android → verify iOS and Web get premium access
- [ ] Test scenario: User purchases on Web → verify iOS and Android get premium access
- [ ] Verify entitlement state updates within 30 seconds across platforms
- [ ] Test offline scenarios: if phone is offline, watch uses cached entitlement state

### Platform-Specific Implementation Details
#### iOS/watchOS
- [ ] Use Watch Connectivity framework to share entitlement state between iPhone and Watch
- [ ] Ensure Watch Extension and iOS app both import same EntitlementManager shared code
- [ ] Implement background update of entitlement state when iPhone state changes
- [ ] Test complications and standalone Watch app functionality

#### Android (Wear OS if applicable)
- [ ] If Wear OS app exists, implement similar shared entitlement state
- [ ] Use appropriate Android mechanisms for cross-device sync (if applicable)
- [ ] Otherwise, document that Wear OS is out of scope for this phase

#### Web
- [ ] Web already shares entitlement state via Supabase auth.uid
- [ ] Verify web entitlement updates in real-time after phone purchases

## Dependencies
- Wave 1: Entitlement Manager Extension (provides cross-platform entitlement interface)
- Wave 2: Translation Gating (provides limit enforcement logic)
- Wave 3: Feature Gating (provides premium feature restriction logic)
- Phase 51: Subscription Backend (provides webhooks and server-side enforcement)
- Phase 52: Paywall UI (provides purchase flow)

## Success Criteria
- Watch app inherits phone subscription status with no separate purchase required
- Watch app respects same feature gating and limits as phone (3 translations/day, last 10 history, premium feature gates)
- Cross-platform entitlement sync works: purchase on any platform activates premium on others within 30 seconds
- All platforms show consistent entitlement state and UI gating
