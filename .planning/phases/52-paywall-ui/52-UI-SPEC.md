## Platform-Specific Visual Contracts

### iOS (RevenueCatUI Template)

| Aspect | Contract |
|--------|----------|
| Paywall view | RevenueCatUI `PaywallView` presented as sheet from Settings row tap (D-03) |
| Visual customization | Limited — configured via RevenueCat dashboard, not code |
| Store compliance | StoreKit-only, no external payment links, no "cheaper on web" text (PAY-06, Pitfall 1) |
| Restore button | Included in RevenueCatUI template by default (PAY-07) |
| Settings row | Standard `UITableView` cell with disclosure indicator, added as new row (existing pattern in `SettingsViewController.swift`) |

### Android (RevenueCatUI Template)

| Aspect | Contract |
|--------|----------|
| Paywall view | RevenueCatUI `Paywall` composable, navigated to from Settings row tap (D-04) |
| Visual customization | Limited — configured via RevenueCat dashboard |
| Restore button | Included in RevenueCatUI template by default (PAY-07) |
| Settings row | New `Row` in `SettingsScreen.kt` Column, consistent with existing `Row` pattern (switch rows) |

### Web (Custom `/subscribe` Page)

| Aspect | Contract |
|--------|----------|
| Route | `/subscribe` — Next.js app route at `web/src/app/subscribe/page.tsx` (D-05) |
| Purchase flow | RevenueCat hosted checkout (Stripe) opens in new browser tab (D-06) |
| Return handling | `customerInfoUpdated` listener in `EntitlementProvider` detects entitlement after user returns from checkout tab; page polls `useEntitlementStore.isPremium` with 3-second interval, max 2 minutes |
| Navigation | Link from Settings page to `/subscribe` (D-05) |
| Deep link | `/subscribe` URL is shareable/deep-linkable |

### Web Interaction States

| State | Visual |
|-------|--------|
| Default | Both plans shown, neither pre-selected |
| Plan selected | Clicking a plan card highlights its border with primary color and scrolls CTA into focus |
| CTA pressed | Button shows loading spinner + "Verifying subscription..." text, button disabled |
| Purchase confirmed | `useEntitlementStore.isPremium` becomes true, redirect to `/settings` |
| Purchase failed | Button re-enables, show error text below button: "Purchase couldn't be completed. Please try again or restore purchases." |
| Offerings unavailable | Show empty state instead of plan cards: heading "Subscription Unavailable", body "We couldn't load subscription options. Check your connection and try again.", retry button |
| Checkout tab open | Show message on `/subscribe`: "Complete your purchase in the other tab. This page will update automatically." with spinner |

## Settings Row Specifications

### iOS Settings Row

| Element | Spec |
|---------|------|
| Row position | New row in `SettingsViewController` table, after existing rows (add as case 7) |
| Cell text | "Subscription" (system font 17px regular) |
| Cell accessory | Disclosure indicator (`cell.accessoryType = .disclosureIndicator`) for free users; detail text "Pro" or "Trial" for premium/trial users |
| Cell detail text color | Green (`systemGreen`) for "Pro"/"Trial" status |
| Tap action | Present `PaywallView` as `.sheet` or `.fullScreenCover` depending on context |
| Auth gate | If `isReadyToAccessPaywall == false`, show sign-in prompt instead of paywall |

### Android Settings Row

| Element | Spec |
|---------|------|
| Row position | New `Row` in `SettingsScreen.kt` Column, below existing rows |
| Row text | "Subscription" (`MaterialTheme.typography.bodyLarge`) |
| Row action | `Text` showing "Subscribe" (muted) or "Pro"/"Trial" (primary color) on right side |
| Tap action | Navigate to RevenueCatUI `Paywall` composable |
| Auth gate | Same as iOS — check `isReadyToAccessPaywall` before navigating |

### Web Settings Link

| Element | Spec |
|---------|------|
| Link position | New card section in `settings/page.tsx`, after "Account" card |
| Card title | "Subscription" (`text-lg font-semibold`) |
| Free user content | "Subscribe" link to `/subscribe` (`text-primary` color) |
| Premium user content | "Pro plan active" text with green `text-primary` color |
| Trial user content | "Trial active" text with `text-muted-foreground`, expiry date shown |
