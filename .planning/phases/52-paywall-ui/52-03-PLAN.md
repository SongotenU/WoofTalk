<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Client → RevenueCat JS SDK | SDK fetches offerings and opens checkout — RevenueCat is trusted intermediary |
| Client → Stripe hosted checkout | Payment processed on Stripe's domain — WoofTalk never sees card details |
| Client → EntitlementProvider | Zustand store state from CustomerInfo listener — UX-only, server is hard gate |
| New tab checkout → WoofTalk tab | Entitlement state propagated via polling + customerInfoUpdated listener |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-52-11 | S (Spoofing) | Entitlement store local state | accept | Server-side RLS (Phase 51) is the hard gate; local state is UX-only |
| T-52-12 | T (Tampering) | Hosted checkout URL | mitigate | Checkout URL comes from RevenueCat SDK (getOfferings response), not user input; window.open target is _blank |
| T-52-13 | I (Information Disclosure) | Subscription status in Settings card | accept | Subscription tier is not PII; visible only to authenticated user |
| T-52-14 | D (Denial of Service) | Polling fails to detect entitlement | mitigate | 2-minute max polling timeout with error state; customerInfoUpdated listener is the primary detection path; polling is fallback |
| T-52-15 | E (Elevation of Privilege) | Client-side entitlement bypass | accept | Server-side RLS (Phase 51 SUB-08/SUB-09) prevents API-level bypass; client display is cosmetic |
| T-52-16 | S (Spoofing) | Checkout tab phishing | mitigate | Checkout opens RevenueCat-hosted Stripe page — user can verify URL in the new tab; no iframe embedding |
</threat_model>
