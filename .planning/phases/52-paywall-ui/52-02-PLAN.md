Create `PaywallScreen.kt` composable that wraps the RevenueCatUI `Paywall` composable and handles the purchase confirmation flow per PAY-08, D-07, D-08:

```kotlin
package com.wooftalk.ui.screen

import androidx.compose.runtime.Composable
import androidx.compose.runtime.collectAsState
import androidx.compose.runtime.getValue
import androidx.compose.ui.platform.LocalContext
import com.revenuecat.purchases.ui.revenuecatui.Paywall
import com.revenuecat.purchases.ui.revenuecatui.PaywallListener
import com.revenuecat.purchases.ui.revenuecatui.PaywallOptions
import com.wooftalk.EntitlementManager

@Composable
fun PaywallScreen(
    entitlementManager: EntitlementManager,
    onDismiss: () -> Unit
) {
    // PAY-08 / D-07: Observe entitlement state to detect purchase confirmation
    val isPremium by entitlementManager.isPremium.collectAsState()
    val isLoading by entitlementManager.isLoading.collectAsState()

    Paywall(
        options = PaywallOptions.Builder()
            .setListener(object : PaywallListener {
                override fun onPurchaseCompleted(customerInfo: CustomerInfo) {
                    // D-08: Refresh entitlements then dismiss
                    // EntitlementManager's UpdatedCustomerInfoListener already fires
                    // from the SDK, but explicitly refresh to be safe
                    kotlinx.coroutines.MainScope().launch {
                        entitlementManager.refreshEntitlements()
                        onDismiss()
                    }
                }

                override fun onRestoreCompleted(customerInfo: CustomerInfo) {
                    kotlinx.coroutines.MainScope().launch {
                        entitlementManager.refreshEntitlements()
                        onDismiss()
                    }
                }
            })
            .build()
    )

    // If entitlement became premium (e.g., from CustomerInfoListener update),
    // dismiss the paywall automatically
    if (isPremium && !isLoading) {
        onDismiss()
    }
}
```

Key implementation notes:
- The `Paywall` composable from `purchases-ui:9.9.0` handles: product display (PAY-04), purchase flow via Play Billing (PAY-02), restore purchases button (PAY-07), loading state during purchase (PAY-08), and offerings verification before display (PAY-09)
- The `PaywallListener.onPurchaseCompleted` callback fires after StoreKit purchase + receipt validation completes — this is the confirmation signal for PAY-08
- "Save 33%" badge on annual plan (PAY-05) is configured in the RevenueCat dashboard, not in code — the template renders it from the offering configuration
- 7-day free trial display (PAY-04) is also driven by RevenueCat offering configuration
- The `isPremium && !isLoading` auto-dismiss handles the case where the `UpdatedCustomerInfoListener` fires before `onPurchaseCompleted` (race condition safety per D-08)

| Client → RevenueCat SDK | Payment processing via Play Billing — RevenueCat is trusted intermediary |
