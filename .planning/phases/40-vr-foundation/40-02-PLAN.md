From 40-RESEARCH.md -- BillboardVR pattern:
```csharp
public class BillboardVR : MonoBehaviour {
    [SerializeField] private Transform cameraTarget; // OVRCameraRig centerEyeAnchor
    void LateUpdate() {
        if (cameraTarget != null) {
            Vector3 flatTarget = new Vector3(
                cameraTarget.position.x,
                transform.position.y,
                cameraTarget.position.z
            );
            transform.LookAt(flatTarget);
        }
    }
}
```

```csharp
public class PinchDetect : MonoBehaviour {

    [SerializeField] float pinchThreshold = 0.8f;

    }
}
```

```csharp
public class BubbleManager : MonoBehaviour {
```

Antipatterns to avoid (from 40-RESEARCH.md):
- Using Update() for billboard causes one-frame jitter -- use LateUpdate()
- Spawning new bubble GameObjects at runtime causes GC spikes at 90 FPS -- use object pool
- Hand tracking with too many UI elements drops to 30 FPS -- simplified visuals only
- Text too small/unreadable -- minimum Canvas 400px width, high contrast background
