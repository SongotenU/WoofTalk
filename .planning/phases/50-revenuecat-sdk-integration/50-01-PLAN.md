<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| App -> RevenueCat API | Client-side SDK communication over HTTPS. API key is public (embedded in app binary). |
| RevenueCat -> App (delegate) | CustomerInfo delivered via delegate callback. Could be tampered on jailbroken devices. |
| App -> Supabase Auth | Auth state determines when logIn is called. Auth token is sensitive. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-50-01 | S (Spoofing) | RevenueCatManager | accept | Client-side SDK init uses public API key only — no secrets exposed. Server-side verification in Phase 51 is the hard gate. |
| T-50-02 | T (Tampering) | EntitlementManager | mitigate | All views read from single EntitlementManager (D-04). Server-side RLS enforces in Phase 51. Client is UX layer only. |
| T-50-03 | I (Info Disclosure) | RevenueCatManager.logIn | accept | auth.uid is not PII — it is a UUID. No sensitive data transmitted beyond what RevenueCat already requires. |
| T-50-04 | E (Elevation) | EntitlementManager.isPremium | mitigate | Offline cache trusted per D-05 (low-risk decision). Server-side enforcement in Phase 51 prevents actual free-tier bypass. |
</threat_model>
