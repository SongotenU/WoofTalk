## Decisions to Remember

- `WatchSyncManager` uses `WCSession.default.updateApplicationContext()` not `sendMessage()` (fire-and-forget is fine for entitlement sync)
- Android WearOS inherits phone subscription via shared RevenueCat identity — no separate Wearable code needed
- Web admin dashboard uses `useEntitlements()` hook (not `useEntitlementSync`) to read server-authoritative state
- All platforms share `Supabase auth.uid` as RevenueCat `appUserID`
