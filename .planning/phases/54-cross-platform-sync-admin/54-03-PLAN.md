# Plan 54-03: Web — Cross-Platform Sync + Admin Subscription Dashboard

**Phase**: 54 — Cross-Platform Sync & Admin
**Scope**: Web app + Admin dashboard
**Depends on**: Phase 50 (RevenueCat SDK), Phase 52 (Paywall UI), Phase 51 (Subscription Backend)

## Goal

Ensure Web entitlements sync in real-time across devices via Supabase, users can manage subscriptions via Stripe portal, and admins can monitor subscription health.

## Tasks

### 1. Add Stripe portal link to Settings page
- In the Subscription section of `/settings`, add a "Manage Subscription" link for premium users
- Link opens Stripe customer portal via RevenueCat's `getWebPurchaseURL()` or direct Stripe portal URL
- For free users: show "View Plans" link to `/subscribe` (already exists)

### 2. Add Supabase real-time entitlement sync
- Create `useEntitlementSync` hook that subscribes to `subscription_status` table changes via Supabase real-time
- On `INSERT` or `UPDATE` to `subscription_status` where `user_id = auth.uid()`: update entitlement store
- Also poll `Purchases.getCustomerInfo()` on window focus to catch changes from other tabs/devices
- Combine with existing `useEntitlementStore` for unified state

### 3. Add subscription metrics to Admin Dashboard
- Add subscription-specific metric cards to `/admin` page:
  - Active Subscribers (count from `subscription_status` where `tier = 'pro'`)
  - Active Trials (count where `tier = 'trial'`)
  - Churn rate (cancellations in last 30d / active at start of period)
- Add "Subscriptions" quick action card linking to new subscriptions page

### 4. Add Admin Subscriptions page
- Create `/admin/subscriptions/page.tsx`
- Table with columns: Email, Tier (Pro/Trial/Free), Platform (iOS/Android/Web), Started, Expires, Status (Active/Cancelled/Expired)
- Filter by tier and status
- Data from `subscription_status` + `user_profiles` joined via Supabase service role

### 5. Add subscription column to Admin Users page
- In `/admin/users`, add a "Subscription" column showing tier badge (Pro/Trial/Free)
- Join `subscription_status` data when fetching users

### 6. Enable RevenueCat Analytics Dashboard
- Document that RevenueCat dashboard should be enabled at `app.revenuecat.com`
- Add analytics link to admin dashboard quick actions

## Verification

1. Settings page shows "Manage Subscription" for premium users, opens Stripe portal
2. Entitlement store updates within 30 seconds of purchase on another platform (via real-time + poll)
3. Admin dashboard shows subscription metrics (subscribers, trials, churn)
4. Admin subscriptions page lists all users with tier, platform, status
