# Plan 54-02: Android — Cross-Platform Sync + Restore + Manage Subscription

**Phase**: 54 — Cross-Platform Sync & Admin
**Scope**: Android app
**Depends on**: Phase 50 (RevenueCat SDK), Phase 52 (Paywall UI)

## Goal

Ensure Android subscription entitlements sync across devices, restore purchases works, and users can manage subscriptions via Play Store link.

## Tasks

### 1. Add Restore Purchases to SettingsScreen
- Add a "Restore Purchases" clickable row below the Subscription row
- On tap: call `Purchases.sharedInstance.restorePurchases()`, then `entitlementManager.refreshEntitlements()`
- Show Snackbar on success/failure

### 2. Add Manage Subscription link to SettingsScreen
- Add a "Manage Subscription" clickable row below Restore Purchases
- Only visible when `entitlementManager.isPremium.value == true`
- On tap: open Play Store subscriptions via `Intent(ACTION_VIEW, Uri.parse("https://play.google.com/store/account/subscriptions"))`

### 3. Add entitlement listener in Application class
- Verify `EntitlementManager` already registers `UpdatedCustomerInfoListener` on init (it does)
- Add `checkEntitlements()` call in `onResume` of main activity to ensure fresh state after cross-platform purchase
- No new Wearable code needed — WearOS inherits from phone via shared RevenueCat identity (same auth.uid)

### 4. Ensure logIn(auth.uid) on every launch
- Already implemented: `EntitlementManager.logIn(authUid)` called after Supabase auth resolves
- Verify `MainApplication` or auth flow calls this on every cold start and warm resume
