  <action>Create `supabase/functions/entitlement-webhook/index.ts` following the existing Edge Function pattern but with these critical differences from standard functions:

**Authentication (D-02):** This function does NOT use `validateAuth` from middleware. Instead, it verifies the incoming `Authorization` header against the `REVENUECAT_WEBHOOK_AUTH` environment variable:
```typescript
const authHeader = req.headers.get('Authorization');
const webhookSecret = Deno.env.get('REVENUECAT_WEBHOOK_AUTH');
if (!authHeader || authHeader !== `Bearer ${webhookSecret}`) {
  return new Response(JSON.stringify({ error: 'Unauthorized' }), {
    status: 401, headers: { ...corsHeaders, 'Content-Type': 'application/json' },
  });
}
```

**Structure in this exact order:**

1. **Imports:**
```typescript
import { serve } from 'https://deno.land/std@0.168.0/http/server.ts';
import { createClient } from 'https://esm.sh/@supabase/supabase-js@2';
import { corsHeaders } from '../_shared/middleware.ts';
```
Do NOT import from subscription.ts — the webhook handles raw data and does its own tier mapping.

2. **Store mapping helper function:**
```typescript
function mapStore(store: string): 'ios' | 'android' | 'web' | 'none' {
  switch (store) {
    case 'APP_STORE': return 'ios';
    case 'PLAY_STORE': return 'android';
    case 'STRIPE': return 'web';
    default: return 'none';
  }
}
```

3. **Main serve handler with this flow:**
   a. OPTIONS preflight: `if (req.method === 'OPTIONS') return new Response('ok', { headers: corsHeaders });`
   b. POST-only check: reject GET/PUT/DELETE with 405
   c. **Authorization header verification** (D-02) as shown above
   d. Create Supabase service role client
   e. Parse request body: `const body = await req.json(); const event = body.event;`
   f. Extract `eventId = event.id`, `eventType = event.type`, `appUserId = event.app_user_id`
   g. **Idempotency check** (SUB-04): query webhook_events for event_id. If exists, return 200 with `{ status: 'duplicate' }`
   h. **Event type handling** (D-01): use a switch statement on eventType covering ALL 14 known types:
      - `INITIAL_PURCHASE`: tier = `event.period_type === 'TRIAL' ? 'trial' : 'pro'`; trialEndsAt from `event.expiration_at_ms`; platform from mapStore
      - `RENEWAL`: tier = `'pro'`; platform from mapStore; if `event.is_trial_conversion` then tier = `'pro'`
      - `TRIAL_STARTED`: tier = `'trial'`; trialEndsAt from `event.expiration_at_ms`; platform from mapStore
      - `TRIAL_CONVERTED`: tier = `'pro'`; platform from mapStore
      - `CANCELLATION`: tier stays at current (`'pro'` — still active until expiration); store `event.cancel_reason` in cancellationReason; platform from mapStore
      - `EXPIRATION`: tier = `'free'`; store `event.expiration_reason` in cancellationReason; trialEndsAt = null
      - `UNCANCELATION`: tier = `'pro'`; cancellationReason = null
      - `BILLING_ISSUE`, `PRODUCT_CHANGE`, `NON_RENEWING_PURCHASE`, `SUBSCRIPTION_PAUSED`, `SUBSCRIPTION_RESUMED`, `TRANSFER`, `TEST`: no-op (just log the event type for monitoring)
      - `default`: log unknown event type with `console.log(`Unknown webhook event type: ${eventType}`)` — do NOT crash
   i. **Upsert subscription_status** (SUB-05, idempotent): only upsert if the event changes tier (skip for no-ops). Use `supabase.from('subscription_status').upsert({...}, { onConflict: 'user_id' })` with fields: user_id (appUserId), revenuecat_id (appUserId), entitlements (event.entitlement_ids || []), subscription_tier, trial_ends_at, purchase_platform, cancellation_reason
   j. **Record event in webhook_events** for idempotency tracking: `supabase.from('webhook_events').insert({ event_id: eventId, event_type: eventType, app_user_id: appUserId })`
   k. **Always return 200 OK** (SUB-05, RESEARCH Pitfall 2): wrap everything in try/catch. On error, `console.error` the error but still return 200. RevenueCat retries non-200 for 72 hours — this must NOT happen.

4. **Error handling pattern:**
```typescript
try {
  // ... all processing logic ...
} catch (err) {
  console.error('Webhook processing error:', err);
  // Still return 200 to prevent RevenueCat retry
}
return new Response(JSON.stringify({ status: 'ok' }), {
  status: 200, headers: { ...corsHeaders, 'Content-Type': 'application/json' },
});
```
The return 200 MUST be outside the try/catch so it always executes.

5. **Variable initialization:** Declare `tier`, `trialEndsAt`, `platform`, `cancellationReason` BEFORE the switch statement with defaults:
```typescript
let tier: 'free' | 'trial' | 'pro' = 'free';
let trialEndsAt: string | null = null;
let platform: 'ios' | 'android' | 'web' | 'none' = 'none';
let cancellationReason: string | null = null;
let shouldUpsert = false;
```
Set `shouldUpsert = true` only in event type branches that change subscription state (INITIAL_PURCHASE, RENEWAL, TRIAL_STARTED, TRIAL_CONVERTED, CANCELLATION, EXPIRATION, UNCANCELATION). Skip upsert for no-op events.

6. **expiration_at_ms conversion:**
```typescript
trialEndsAt = event.expiration_at_ms
  ? new Date(event.expiration_at_ms).toISOString()
  : null;
```
