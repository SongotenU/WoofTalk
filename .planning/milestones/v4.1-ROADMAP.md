# Milestone v4.1: Security & Deployment Hardening

**Status:** ✅ SHIPPED 2026-04-02
**Phases:** 33-37
**Total Plans:** 5

## Overview

Close security gaps from v4.0 execution — admin route guards, API IP allowlisting, consumer regression testing, email invites, and E2E verification against live Supabase. Validates the enterprise platform for production deployment.

## Milestone Summary

**Key Decisions:**
- Admin auth via middleware + requireAdmin() helper (defense in depth)
- IP allowlisting is optional per API key (empty = no restriction)
- Consumer regression tested via shell script, not framework tests

**Technical Debt Incurred:**
- v4.1 requirements (SEC-AUTH-01, SEC-AUTH-02, etc.) not formally defined in REQUIREMENTS.md
- E2E tests (Phase 37 success criteria 1) not run against live Supabase deployment
